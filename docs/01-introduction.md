# 01. 项目介绍与背景 (Introduction)

## 为什么会有 TouchDX？

在 PC 端游玩街机移植类音游（特别是 **maimai DX (舞萌 DX)**）时，一直存在一个难以逾越的障碍：**操控还原度**。

传统的键盘、手柄或鼠标，完全无法重现 maimai DX 那块标志性的圆形触摸屏和外围八个实体按键的交互体验。许多玩家尝试使用 Windows 平板或带触摸屏的笔记本电脑进行游玩，却立刻遭遇了另一个致命的打击：**Windows 操作系统的默认触摸手势**。

当你用多根手指在屏幕上快速滑动、点击时，Windows 经常会将这些操作识别为：
*   呼出多任务视图
*   切换桌面
*   页面滚动
*   长按右键菜单

这些系统级的判定拥有极高的优先级，会导致游戏窗口失去焦点、判定丢失，严重破坏打歌体验。

## TouchDX 解决方案

**TouchDX** 是一套彻底解决上述痛点的软硬件桥接方案。

我们放弃了“让 Windows 更好地理解触摸”这一错误的方向，转而采用一种更极客、更原生的思路：
**将另一台移动设备（如 Android 平板或手机），完全改造为一个专属于 maimai DX 的原生触控外设。**

通过将 Android 客户端与 PC 端游戏解耦：
1.  **在 Android 端**：创建一个纯净的、全屏的应用程序。Android 完善的多点触控 API 能够毫无保留、毫无干扰地捕捉每一次点击和滑动。
2.  **在传输层**：通过 ADB (Android Debug Bridge) 有线连接，建立一条极低延迟的 TCP 通道。
3.  **在 PC 端**：通过 MelonLoader 和 Harmony 技术，直接从内存层面将接收到的触摸数据强行“注入”到游戏的输入管理模块中。

## 目标与愿景

TouchDX 的终极目标是：**在不购买昂贵专用框体控制器的前提下，利用玩家现有的移动设备，提供最接近街机原生的、0 延迟的打歌体验。**

本系列文档将为你详细拆解这套系统是如何运作的，并指导你如何配置、使用以及进行二次开发。
