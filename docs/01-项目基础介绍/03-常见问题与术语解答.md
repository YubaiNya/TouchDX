# 03. 常见问题与术语解答

在接触 TouchDX 及其架构时，您可能会遇到一些专有名词。本篇文档将为您解答常见的疑问。

## 术语解释

*   **ADB (Android Debug Bridge)**：Android 调试桥。开发者用来与安卓设备通信的命令行工具。TouchDX 借用它的底层通道传输触摸数据，是实现 0 延迟的核心。
*   **MelonLoader**：一个通用的 Unity 游戏 Mod 加载器。由于 maimai DX 是基于 Unity 引擎构建的，我们需要通过它将我们的定制代码挂载到游戏进程中。
*   **Harmony**：一个用于 .NET 环境的强大的内存劫持库。它允许我们在不修改游戏原始文件的情况下，在游戏代码执行前（Prefix）或执行后（Postfix）插入我们自己的逻辑。
*   **SVG (Scalable Vector Graphics)**：可缩放矢量图形。与像素图（如 PNG、JPG）不同，它是基于数学公式绘制的。这使得它非常适合用来精确定义 maimai DX 面板上各种不规则的按键形状。

## 常见 Q&A

**Q: 为什么我用 Wi-Fi 玩起来感觉很卡，经常断触？**
A: Wi-Fi 协议在设计时优先考虑的是传输的“可靠性”和“吞吐量”，而不是“实时性”。空气中的电磁干扰、路由器的缓冲队列都会导致数据包到达的时间忽快忽慢（即网络抖动 Jitter）。在 60 帧的游戏中，一帧只有 16 毫秒，Wi-Fi 几十毫秒的波动足以让你错过 Perfect 判定甚至造成 Slide 断连。因此，**必须使用 USB 线 + ADB 转发**。

**Q: 游戏更新了，TouchDX 还能用吗？**
A: 这取决于游戏更新是否修改了底层的输入逻辑结构（如重写了 `MeshButton` 类）。只要底层的命名空间和类名没有发生根本性变动，TouchDX-Mod 依然可以通过 Harmony 自动挂载。如果发生大版本更新，可能需要开发者重新定位函数签名并更新 Mod。
