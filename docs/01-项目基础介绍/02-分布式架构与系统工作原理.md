# 02. 分布式架构与系统工作原理

TouchDX 不是一个单一的软件，而是一套分布式的系统。为了实现从物理触摸到游戏内判定的毫秒级同步，我们将其拆分为三个核心子项目，外加一条高速通信总线。

## 架构概览

```text
[ 安卓设备 (平板/手机) ]                    [ PC (Windows) ]
       |                                          |
       |                                          v
+------------------+                    +---------------------+
|   TouchDX App    |                    |  maimai DX (游戏)   |
|  (MaiNative)     |                    |                     |
|                  |     网络通信        |  +---------------+  |
| - SVG 解析与渲染  | -----------------> |  | TouchDX-Mod   |  |
| - 原生触摸事件处理|   (ADB 有线转发)     |  | (MelonLoader) |  |
| - 按键状态位掩码  |  TCP Port: 4321    |  +---------------+  |
+------------------+                    +---------------------+
                                                  ^
                                                  | 
                                                  v
                                        +---------------------+
                                        | PCDiagnosticTool    |
                                        | (独立诊断抓包工具)    |
                                        +---------------------+
```

## 核心组件解析

### 1. MaiNative (安卓客户端)
这是玩家直接交互的终端。它使用 Android 原生的 SDK 开发，保证了最低的触控响应延迟和最高的硬件兼容性。
*   **输入捕获**：以最高帧率捕获屏幕的所有触控点（Pointer）。
*   **空间换算**：将触摸坐标与 SVG 定义的不规则几何图形进行相交测试。
*   **状态打包**：将最终的判定结果打包为 64 位的掩码发送给 PC。

### 2. TouchDX-Mod (PC 注入端)
这是系统的“大脑中枢”。它是一个运行在 MelonLoader 环境下的动态链接库 (`.dll`)。
*   **网络监听**：独立线程在 `4321` 端口开启 Socket 监听。
*   **状态反序列化**：接收并解析 64 位掩码。
*   **游戏干涉**：根据掩码，分别调用 `user32.dll` 模拟硬件键盘，并调用 Harmony 补丁直接修改游戏内存中的 `MeshButton` 状态。

### 3. 通信总线：ADB 反向代理
为了解决传统 Wi-Fi 的 Jitter (网络抖动) 导致的判定不稳定，我们利用了 Android 开发者工具中的 `adb reverse`。
它通过 USB 数据线，将 PC 的本地端口反向映射到安卓设备上。这样不仅拥有了超越千兆网卡的物理带宽，还彻底规避了路由器的排队延迟。
