# 08. 常见问题排查与诊断 (Troubleshooting)

在配置 TouchDX 的过程中，由于涉及到跨平台网络通信和 Mod 注入，难免会遇到一些连接或判定失效的问题。请参考此文档进行自助排查。

## 万能排查利器：PCDiagnosticTool

我们提供了一个专门的抓包检测工具 `TouchDX-DiagnosticTool`。它可以完全模拟游戏服务端的网络行为，直观展示手机端发送过来的所有数据。

**使用步骤**：
1. 确保游戏（maimai DX）处于 **关闭状态**（否则 4321 端口会被占用）。
2. 运行 `PCDiagnosticTool.exe`。
3. 此时控制台会显示正在监听 `4321` 端口。
4. 确保你已经执行了 `adb reverse tcp:4321 tcp:4321`。
5. 在手机 App 填入 `127.0.0.1` 并点击连接。
6. 如果控制台显示 `Android Client Connected`，并能随着你拍打屏幕实时刷出 `[TouchA1] [Btn3]` 这样的文字，**说明网络层和手机端完全正常。问题必定出在游戏和 Mod 端。**

## 常见 Q&A

### Q1: 手机点击连接后一直显示“失败”或没有任何反应？
1.  **检查 ADB 状态**：在 cmd 输入 `adb devices` 必须显示设备状态为 `device` 而不是 `offline` 或 `unauthorized`。
2.  **重新映射端口**：每次拔插数据线、或者手机断开连接后，`adb reverse tcp:4321 tcp:4321` 这条命令都必须 **重新执行一遍**。
3.  **检查游戏是否开启**：必须先打开游戏（或诊断工具）让服务端启动，然后手机端再去点击连接。

### Q2: 诊断工具能看到触摸数据，但在游戏里按屏幕没有任何反应？
1.  **Mod 加载失败**：检查游戏启动时 MelonLoader 的黑色控制台窗口。查看是否有一行绿字 `Starting MaiRemoteTouch TCP Server on port 4321...` 和 `Harmony Patches applied!`。如果没有，说明 Mod 放错了位置，或者 MelonLoader 版本不兼容。
2.  **游戏焦点丢失**：确保游戏的窗口处于 Windows 的最顶层激活状态。如果使用外接显示器或截屏软件，可能会导致焦点丢失，从而无法响应键盘按键。

### Q3: A 区触摸在选歌菜单没反应，但在打歌的时候正常？
这是**正常设定**。为了方便选歌，在非局内状态下，A 区的触摸会被代码强制转换并映射为外围实体物理按键 1-8。一旦歌曲开始，A 区会自动恢复为正常的屏幕触摸判定。

### Q4: 延迟时高时低，手感很怪（甚至断连）？
1.  **你一定是在用 Wi-Fi**！请立刻换回 USB 数据线并使用 ADB 转发。
2.  如果已经在使用有线连接，请检查数据线是否有内部破损导致接触不良，建议更换原装高质量数据线。
3.  检查电脑是否有占用极高 CPU 资源的后台进程（如杀毒软件扫描、系统更新等），导致网络处理线程卡顿。
