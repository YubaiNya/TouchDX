# 07. 使用诊断工具排查网络故障

有时候按照教程操作了，但进游戏点屏幕还是没反应。为了避免“盲人摸象”，我们开发了 `PCDiagnosticTool`。这是一款独立的抓包诊断工具。

## 原理
`PCDiagnosticTool` 模拟了游戏 Mod 的行为。它同样会在 `4321` 端口建立 TCP 服务器。如果安卓客户端能连上它，并能成功发送按键数据，就排除了“网络、数据线、安卓 App、手机系统”这四方的问题，证明只剩下“游戏本体或 Mod 注入”环节出了差错。

## 使用方法

1. **非常重要**：首先请**彻底关闭 maimai DX 游戏程序**。因为端口不能被两个程序同时占用。
2. 前往 [TouchDX-DiagnosticTool Releases](https://github.com/YubaiNya/TouchDX-DiagnosticTool/releases) 下载最新的 `.exe` 程序。
3. 双击运行，弹出一个黑框。上面应该显示 `Listening on port 4321...`。
4. 按照之前的教程，用 USB 线连好手机，并执行一次 `adb reverse tcp:4321 tcp:4321`。
5. 打开手机上的 TouchDX，IP 填入 `127.0.0.1` 并点击连接。

## 观察现象

### 现象 A：手机连接失败
*   **诊断**：手机根本找不到电脑。
*   **排查方案**：
    *   你真的执行了 `adb reverse` 吗？有没有敲错命令？
    *   `adb devices` 能看到设备吗？
    *   是不是没关游戏？导致 4321 端口被游戏占了，诊断工具没启动成功？

### 现象 B：手机连接成功，但拍打屏幕黑框里没数据
*   **诊断**：网络通了，但安卓客户端没有采集到触摸。
*   **排查方案**：
    *   检查手机 App 的设置，是不是按到了“锁定触摸 (穿透到桌面)”导致没有采集输入？

### 现象 C：手机连接成功，拍打屏幕时黑框疯狂刷新文字
*   屏幕上会出现类似：
    `Raw Mask: 0x0000000000000100`
    `Inputs: [TouchA1]`
*   **诊断**：**恭喜你，你的网络环境、数据线、安卓客户端堪称完美。** 没有任何问题。
*   **排查方案**：此时如果进游戏没反应，请 100% 确认你是否正确安装了 MelonLoader，并且 `MaiRemoteTouchMod.dll` 被放在了正确的位置。检查游戏的控制台输出是否有红色的 Error 报错。

## 附：隐藏的 Ping 延迟测试
当连接到诊断工具后，手机端的悬浮窗会显示黄色的 **“测试模式”** 字样。
此时悬浮窗会解锁一个 **“测网络延迟”** 的按钮。点击它，客户端会向诊断工具发包并计算往返时间（RTT）。你可以用这个功能来直观感受一下你家的 Wi-Fi 延迟抖动有多么可怕，从而坚定使用 ADB 有线的决心。
