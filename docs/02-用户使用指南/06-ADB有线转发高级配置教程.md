# 06. ADB 有线转发高级配置教程

这是极其重要的一章。**不要用 Wi-Fi 玩 maimai DX，除非你不在乎你的评分。**

ADB (Android Debug Bridge) 的 `reverse` 功能是连接 Android 手机与电脑内网通信的魔法。它可以通过物理的 USB 数据线，建立一条从 PC 端口到 Android 端口的高速、无干扰、0 延迟的隧道。

## 完整配置步骤

### 1. 准备 ADB 环境
1. 下载官方的 [SDK Platform-Tools](https://developer.android.com/studio/releases/platform-tools)。
2. 解压到一个你找得到的地方（例如 `C:\adb`）。
3. 将这个路径 (`C:\adb`) 添加到 Windows 的“系统环境变量 -> Path”中。这样你就可以在任何地方打开 CMD 使用 `adb` 命令了。

### 2. 手机开启 USB 调试
1. 拿起你的安卓设备，进入“设置 -> 关于手机”。
2. 连续点击“版本号”或“OS版本” 7 次，直到提示“您已处于开发者模式”。
3. 退回设置菜单，找到“开发者选项”。
4. 打开**“USB 调试”**开关。

### 3. 建立连接
1. 使用一根**质量好、能传输数据**的 USB 线将设备连接到电脑。
2. 此时手机屏幕可能会弹出一个提示框：“是否允许 USB 调试？”，勾选“一律允许使用这台计算机进行调试”，点击**确定**。
3. 在电脑上按下 `Win + R` 键，输入 `cmd` 回车，打开命令行窗口。
4. 输入验证命令：
   ```bash
   adb devices
   ```
5. 正常情况下，应该输出类似这样的一行：
   `XXXXXXXXXXXX    device`
   *(如果是 offline 或 unauthorized，说明授权失败，拔插线重试并留意手机弹窗。)*

### 4. 开启魔法转发通道
在确认设备为 `device` 状态后，继续在 CMD 中输入：
```bash
adb reverse tcp:4321 tcp:4321
```
按下回车。如果**没有任何多余的报错输出，直接跳到下一行**，就说明转发成功了！

> **原理说明**：这条命令的意思是，在安卓设备的系统底层监听 4321 端口。当安卓 App 尝试访问 `127.0.0.1:4321` 时，Android 系统会将这些数据顺着 USB 线，直接投递给 PC 电脑本机的 `4321` 端口。而 PC 端的 TouchDX Mod 刚好在这个端口等待接收！这就是为什么在安卓 App 里填入 `127.0.0.1` 就能直接连上电脑的原因。
