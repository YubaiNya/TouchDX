# 06. 自定义 SVG 布局教程 (Custom SVG Layout)

TouchDX 最大的灵活性在于，它将界面的按键布局完全交给了玩家。只需修改一个 SVG (可缩放矢量图形) 文件，你就可以任意调整按键的位置、大小、形状。

## 文件位置
SVG 文件被打包在 Android APK 中。如果你要进行修改，需要获取 `MaiNative` 的源码，修改 `assets/maimai-layout.svg`，然后重新编译打包。

## SVG 的结构要求

`SvgLoader.java` 对 SVG 文件的解析有特定的规则：

1.  **路径元素 (`<path>`)**：系统只会解析带有完整 `d="M... C... Z"` 属性的 `<path>` 元素。如果你在作图软件中画了圆形 (`<circle>`) 或矩形 (`<rect>`)，请务必将其“转换为路径”。
2.  **ID 命名规范**：这是非常重要的一步。每个路径必须有一个 `id` 属性，`SvgLoader` 会根据这个 ID 来判断它代表哪个按键。

合法的 ID 命名包括：
*   `A1` 到 `A8`
*   `B1` 到 `B8`
*   `C1`, `C2`
*   `D1` 到 `D8`
*   `E1` 到 `E8`
*   `Btn1` 到 `Btn8` (外围实体按键)
*   `Select`, `Test`, `Service`, `Coin`

> 注意大小写敏感。如果在解析时发现某个 `id` 匹配上述规则，那么当玩家触摸这个区域时，网络掩码的对应 bit 位就会被置为 1。

## 修改流程

1.  打开矢量图形设计软件（如 Adobe Illustrator, Inkscape 等）。
2.  新建一个尺寸为你手机屏幕分辨率（或等比例）的画布，通常我们以设备横屏状态设计。
3.  根据 maimai DX 的机台面板图纸或按照你个人的喜好，绘制所有的按键区域。
4.  确保按键之间没有重叠，除非你有意为之（重叠区域被触摸时，两个按键都会被判定为按下）。
5.  在图层面板中，为每个路径图层重命名为上述的合法 ID（如 `A1`）。
6.  将画布导出为 SVG 代码（建议选择精简/压缩模式，去除多余的 CSS 和私有标签）。
7.  覆盖 `assets/maimai-layout.svg` 文件。
8.  使用 Android Studio 重新构建你的专属 `TouchDX` 安装包。
