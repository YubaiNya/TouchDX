# 05. 网络通信与数据协议 (Network Protocol)

为了确保绝对的低延迟，TouchDX 摒弃了 HTTP、WebSocket 或任何带有复杂握手和解析逻辑的上层协议，直接使用了底层的 **TCP Socket**。

## 端口与连接
*   **端口号**: 默认固定为 `4321`。
*   **Server 端**: PC 端的 Mod 和 DiagnosticTool 作为服务端，启动后监听 `0.0.0.0:4321`。
*   **Client 端**: Android 客户端作为客户端，主动向指定的 IP 发起连接。

> 为什么不使用 UDP？虽然 UDP 无连接且通常认为延迟更低，但对于精确的按键输入来说，丢失任何一个“抬起(UP)”的数据包都会导致断触或长按卡死。在内网/USB 有线环境下，关闭了 Nagle 算法（`NoDelay = true`）的 TCP 已经能提供微秒级的传输速度，并附带了可靠的数据校验。

## 数据包格式 (Data Packet)

网络中传输的核心数据是一个固定的 **8 字节 (64-bit)** 块，小端序 (Little-Endian)。

这 64 个 bit 的具体映射如下（从低位 Bit 0 到 高位 Bit 63）：

| Bit 范围 | 对应名称 | 映射目标 | 说明 |
| :--- | :--- | :--- | :--- |
| `0-7` | `Btn1` ~ `Btn8` | 物理按键 1-8 | 顺时针排列，1号位在1点半方向 |
| `8-15` | `TouchA1` ~ `TouchA8` | 触摸 A 区 | 屏幕最外圈 |
| `16-23` | `TouchB1` ~ `TouchB8` | 触摸 B 区 | 屏幕中间圈 |
| `24-25` | `TouchC1`, `TouchC2` | 触摸 C 区 | 屏幕最内圈 |
| `26-33` | `TouchD1` ~ `TouchD8` | 触摸 D 区 | 星星滑动专用外圈 |
| `34-41` | `TouchE1` ~ `TouchE8` | 触摸 E 区 | 星星滑动专用内圈 |
| `42` | `Select` | Select 控制键 | 用于测试菜单选择 |
| `43` | `Test` | Test 控制键 | 进入测试模式 |
| `44` | `Service` | Service 控制键 | 菜单服务键 |
| `45` | `Coin` | 投币键 | 用于投币 |
| `46-63` | 保留 | / | 留作后续功能扩展 |

## 握手与心跳

*   **状态同步**：为了让安卓端知道当前游戏的状态（例如是否在局内），PC Mod 在没有触摸状态变化时，会定期（1ms）向安卓发送心跳包。
*   **心跳包格式**：`0x47, 0x41, 0x4D, 0x45, 0, 0, 0, [isInGame]`。前4个字节对应 ASCII 字符 `GAME`，最后一个字节表示当前是否在游戏判定状态中。安卓端据此决定 A 区触摸是映射为触屏事件还是物理按钮事件。
